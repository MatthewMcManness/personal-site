Part 1: Wipe Windows & Merge Partitions

Current Partition Layout

PartitionSizeTypeWhat It Is
nvme0n1p1512MEFI (vfat)Shared boot partition — KEEP
nvme0n1p21.3Text4Linux install (/) — EXPAND THIS
nvme0n1p3140.5Gexfat"Shared" drive — DELETE
nvme0n1p416M—Windows reserved — DELETE
nvme0n1p5402.9GNTFSWindows 11 install — DELETE
nvme0n1p6703MNTFSWindows recovery — DELETE

After merging, Linux will grow from ~1.3T to ~1.8T.

WARNING: This is destructive and irreversible. Power loss during resize = data loss. Make sure your machine is plugged in.

Step 1: Edit fstab (Do This Before Rebooting)

Remove the shared drive mount so Linux doesn't try to mount a deleted partition on boot:

sudo sed -i '18s/^/#/' /etc/fstab

Or manually edit /etc/fstab and add a # at the start of line 18 (the /mnt/shared exfat entry).

Verify:

cat /etc/fstab

Line 18 should now look like:

#UUID=6FEB-374D  /mnt/shared  exfat  defaults,uid=1000,gid=1000,nofail  0  0

Step 2: Boot from a Live USB

You cannot resize your root partition while it's mounted. You need a Linux live USB.

  1. Download an Ubuntu ISO if you don't have one
  2. Flash it to a USB: 
    sudo dd if=ubuntu.iso of=/dev/sdX bs=4M status=progress

    (Replace /dev/sdX with your USB device — check with lsblk. Or use Balena Etcher.)

  3. Reboot and boot from the USB (usually F12 or F2 at boot for the boot menu)
  4. Choose "Try Ubuntu" (don't install)

Step 3: Open GParted

Once in the live environment, open a terminal:

sudo apt install gparted -y   # usually pre-installed
sudo gparted

Step 4: Delete the Windows & Shared Partitions

In GParted, select your NVMe drive (/dev/nvme0n1), then:

  1. Right-click nvme0n1p6 (703M NTFS, Windows recovery) → Delete
  2. Right-click nvme0n1p5 (402.9G NTFS, Windows install) → Delete
  3. Right-click nvme0n1p4 (16M, Windows reserved) → Delete
  4. Right-click nvme0n1p3 (140.5G exfat, Shared) → Delete

You should now see one big block of unallocated space (~544G) to the right of nvme0n1p2.

Step 5: Expand the Linux Partition

  1. Right-click nvme0n1p2 (your 1.3T ext4 Linux partition) → Resize/Move
  2. Drag the right edge all the way to the right to fill all the unallocated space
  3. Click Resize/Move

Step 6: Apply Changes

Click the green checkmark (Apply All Operations) in the toolbar. This will:

  • Delete the 4 partitions
  • Resize your Linux partition from ~1.3T to ~1.8T

This may take 5–30 minutes depending on disk usage.

Step 7: Fix the Bootloader

Since we deleted Windows, clean up GRUB so it no longer shows the Windows boot entry:

# Mount your Linux partition
sudo mount /dev/nvme0n1p2 /mnt
sudo mount /dev/nvme0n1p1 /mnt/boot/efi
sudo mount --bind /dev /mnt/dev
sudo mount --bind /proc /mnt/proc
sudo mount --bind /sys /mnt/sys

# Chroot into your install
sudo chroot /mnt

# Update GRUB to remove Windows entry
update-grub

# Exit chroot and unmount
exit
sudo umount -R /mnt

Step 8: Reboot

Remove the USB and reboot. Verify with:

df -h /

You should see ~1.8T for your root partition.

Part 2: KVM/QEMU + virt-manager Setup

Do this after you're back in your regular Linux install.

System info: 64 CPU threads, 32GB RAM, KVM virtualization supported. This will run great.

Step 1: Install KVM and Tools

sudo apt update
sudo apt install -y qemu-kvm libvirt-daemon-system libvirt-clients \
  bridge-utils virt-manager ovmf
PackagePurpose
qemu-kvmThe actual VM hypervisor
libvirt-daemon-systemManagement daemon for VMs
libvirt-clientsCLI tools (virsh command)
bridge-utilsNetwork bridging for VM internet access
virt-managerGUI to manage VMs (like VirtualBox's GUI)
ovmfUEFI firmware for VMs (needed for Windows 11)

Step 2: Add Yourself to Required Groups

sudo usermod -aG libvirt,kvm $USER

Log out and back in (or reboot) for group changes to take effect.

Step 3: Verify Everything Works

# Check KVM is working
virsh list --all

# Should show an empty table, no errors

Step 4: Download What You Need

  1. Windows 10 or 11 ISO — Download from Microsoft's website
  2. VirtIO drivers ISO — These give your VM much better disk/network performance: 
    wget https://fedorapeople.org/groups/virt/virtio-win/direct-downloads/stable-virtio/virtio-win.iso \
  -O ~/Downloads/virtio-win.iso

Step 5: Create the VM in virt-manager

  1. Open virt-manager from your app launcher (or run virt-manager in terminal)
  2. Click "Create a new virtual machine" (the monitor icon with a star)
  3. Step 1 — Install media:
    1. Select "Local install media (ISO image)"
    2. Click "Browse" → "Browse Local" → Select your Windows ISO
    3. It should auto-detect "Microsoft Windows 10/11". If not, type it manually.
  4. Step 2 — Memory and CPU:
    1. RAM: 8192 MB (8GB — you have 32GB so this is fine)
    2. CPUs: 8 (you have 64 threads, giving 8 to the VM is plenty)
  5. Step 3 — Storage:
    1. Select "Create a disk image for the virtual machine"
    2. Size: 100 GB (or more if your ROM hack tools need space)
  6. Step 4 — Final settings:
    1. Check "Customize configuration before install"
    2. Name it something like "Windows-ROMHack"
    3. Click Finish

Step 6: Customize Before Installing (Important!)

The configuration window opens. Make these changes:

Overview tab:

  • Firmware: Change to UEFI x86_64: /usr/share/OVMF/OVMF_CODE_4M.fd
    (Required for Windows 11, recommended for Windows 10 too)

CPUs tab:

  • Topology: Check "Manually set" → 1 socket, 4 cores, 2 threads

Disk (VirtIO Disk 1) tab:

  • Disk bus: Change to VirtIO (much faster than the default SATA)

NIC tab:

  • Device model: Change to virtio (faster networking)

Add Hardware → Storage:

  • Select "CDROM device"
  • Browse to your virtio-win.iso
  • This adds a second CD drive with the VirtIO drivers

Click "Begin Installation".

Step 7: Install Windows

The Windows installer will start. When you get to the "Where do you want to install Windows?" screen:

  1. You'll see no drives listed (because Windows doesn't have VirtIO drivers yet)
  2. Click "Load driver"
  3. Browse to the VirtIO CD → viostor → w11 (or w10) → amd64
  4. Select the driver and click Next
  5. Your virtual disk will now appear — select it and continue the install

Step 8: Post-Install — Install VirtIO Guest Tools

After Windows boots:

  1. Open File Explorer
  2. Go to the VirtIO CD drive
  3. Run virtio-win-guest-tools.exe
  4. This installs all remaining drivers (network, display, balloon, etc.)

Step 9: Set Up a Shared Folder

Easiest way to move ROM files between Linux and the Windows VM:

On the Linux side:

mkdir -p ~/vm-shared

In virt-manager (shut down the VM first):

  1. Click "Add Hardware" → Filesystem
  2. Driver: virtiofs
  3. Source path: /home/matthew/vm-shared
  4. Target path: share (this is just a label, not a Windows path)

In the Windows VM:

  1. Install WinFsp from https://winfsp.dev/
  2. The VirtIO FS driver should already be installed from the guest tools
  3. After WinFsp is installed, the shared folder appears as a network drive (usually Z:)
Alternative: If shared folders are finicky, just use a USB drive or drag-and-drop through virt-manager's SPICE display.

Step 10: Quality of Life Tweaks

Clipboard sharing (copy/paste between host and VM):

  • The spice-vdagent should be installed by the VirtIO guest tools
  • If not, install SPICE Guest Tools in Windows from https://www.spice-space.org/download.html

Display resolution:

  • In VM settings, change Video to QXL or Virtio for resizable display
  • The VirtIO guest tools should make the VM display resize with the window

Auto-start the VM (optional):

virsh autostart Windows-ROMHack

Quick Reference Commands

# Start / stop VMs from terminal
virsh start Windows-ROMHack
virsh shutdown Windows-ROMHack        # graceful shutdown
virsh destroy Windows-ROMHack         # force off (like pulling the plug)

# List all VMs
virsh list --all

# Take a snapshot (save state before risky changes)
virsh snapshot-create-as Windows-ROMHack "before-mod" "Clean state"
virsh snapshot-revert Windows-ROMHack "before-mod"

# Open the GUI manager
virt-manager

Summary Checklist

  1. Edit /etc/fstab — comment out the shared mount line
  2. Boot from live USB — use GParted to delete Windows partitions and expand Linux
  3. Update GRUB — remove the Windows boot entry
  4. Install KVM — sudo apt install qemu-kvm libvirt-daemon-system virt-manager ovmf
  5. Add user to groups — sudo usermod -aG libvirt,kvm $USER then re-login
  6. Create VM — use virt-manager with VirtIO drivers for best performance
  7. Set up shared folder — use virtiofs to pass ROM files between host and VM
